### What was broken ‚Äî stacked, in the order they surfaced
1. **Crash on tap ‚Üí missing `NSBluetoothAlwaysUsageDescription`.** The Stripe Terminal SDK aborts at `discoverReaders` (SIGABRT) without a Bluetooth usage string ‚Äî required even for Tap to Pay (shared SDK). Added it (+ `NSBluetoothPeripheralUsageDescription` + `NSLocationWhenInUseUsageDescription`) to `ios/App/App/Info.plist`. **Location string is also mandatory** (Stripe uses location) or it SIGABRTs earlier.
2. **Entitlement not on the App ID.** Apple granted Tap to Pay (Case-ID **20814883**, dev tier) but it hadn't propagated; a hand-edited `App.entitlements` key fails signing ("entitlement not found / could not be included in profile"). ‚úÖ Fix: add it in Xcode via **Signing & Capabilities ‚Üí + Capability ‚Üí "Tap to Pay on iPhone"** ‚Äî that registers it with the App ID + regenerates the profile. Once it shows up under + Capability, the account has it. (It did NOT appear for ~hours after Apple's email ‚Äî propagation lag; "Tap to Share" is a DIFFERENT capability, don't add it.)
3. **Freeze after connect ‚Üí no Stripe Terminal Location.** The plugin force-unwraps `locationId` (`node_modules/@capacitor-community/stripe-terminal/ios/.../StripeTerminal.swift:152`), and it's only set from what the CLIENT passes to `discoverReaders`. ‚úÖ Fix: (a) enabled **Terminal** on the live Stripe account + created a **Location** "Sanctuary Barber Co" (Dashboard ‚Üí Terminal ‚Üí Locations); (b) added a staff-only **`terminal_location`** action in `api/stripe.js` (shared resolver `terminalLocationId()`, also scopes the `connection_token` to the Location); (c) client fetches it and passes `locationId` into `discoverReaders`. Also **hardened `src/tapToPay.js`** ‚Äî timeouts on init/discover/connect/token/intent, surfaces the real error instead of a silent hang, and stops swallowing connection-token failures (captures + hands the SDK an empty token so it fails fast). Added `await ensureFreshSession()` before the tap in `runTapToPay` (backgrounded iOS JWT ‚Üí 401 ‚Üí hang).
4. **‚≠ê‚≠ê THE TIME-SINK ‚Äî stale cached web code on the phone.** The native app loads its UI from **gotvero.com** and WKWebView **caches it**. A web deploy AND a clean Xcode rebuild do **NOT** clear that cache ‚Äî so the phone kept running the OLD `tapToPay.js` (no `locationId`) and crashing at connect, over and over, even though the fix was live on the server and in the repo. **‚úÖ Fix: DELETE the app from the phone and reinstall it** (long-press ‚Üí Remove App ‚Üí Delete App, then ‚ñ∂ in Xcode). That drops the WKWebView cache and forces fresh JS. **RULE: after a web deploy that must reach the native app, delete+reinstall ‚Äî a rebuild alone is not enough.** (Also cost time: a `git reset --hard origin/<branch>` was needed on Dan's Mac because an earlier rebase force-push left his local Info.plist without the Bluetooth key ‚Äî verify `grep -c NSBluetoothAlwaysUsageDescription ios/App/App/Info.plist` ‚Üí 1 before building.)
